# 今日やる / 今後やる / カレンダー 設計メモ

## 方針

今後は「今日かどうか」を専用フラグで持つのではなく、タスクに予定日を持たせる。

予定日から、今日やるもの・今後やるもの・カレンダーに出すものを判定する。

## 基本のタブ

- 今日やる
- 今後やる
- カレンダー

## データ設計

タスクには予定日を持たせる。

```ts
Item {
  id: string
  title: string
  tag: string | null
  recurring: boolean
  scheduledDate: string | null // YYYY-MM-DD。未定なら null
  status: "open" | "done"
  createdAt: string
  doneAt?: string
}
```

現状の `today_items` は `scheduledDate` に修正する。

## 表示条件

### 今日やる

以下を表示する。

- 毎日タスク
- `scheduledDate` が今日以前の未完了タスク
- 今日の日付で追加されたタスク

昨日以前に予定していて未完了のタスクも、今日やるに残る。

### 今後やる

以下を表示する。

- 日付未定の未完了タスク
- `scheduledDate` が明日以降の未完了タスク

毎日タスクは表示しなくてよい。

### カレンダー

以下を表示する。

- 予定日が入っている未完了タスク
- できたことログ
- その日の「ひとこと」

## 今日やる画面

今日やる画面は実行と編集の両方ができる。

- 今日の日付の新しいタスクを追加できる
- 今日の既存タスクを編集できる
- 今日の既存タスクを削除できる
- 毎日タスクも表示する
- 毎日タスクも編集できる
- チェックボックスを表示する
- チェックすると完了状態になる
- 完了すると「できたこと」として今日の日付で記録する
- 完了したタスクは今日やる一覧に残す
- 完了状態は取り消せる
- 小さなステップを追加・削除・完了できる

既存の `/36達成` のような達成数表示は維持する。

## 今後やる画面

今後やる画面は予定整理と編集の場所。

- 日付未定の新しいタスクを追加できる
- 日付指定した新しいタスクを追加できる
- 既存タスクを編集できる
- 既存タスクを削除できる
- 日付を変更できる
- 日付なしにできる
- 今日やるにできる
- 毎日タスクは表示しなくてよい
- チェックボックスは表示しない
- 完了操作はしない

今後やる画面でチェック完了はしない。
未来予定のタスクを今日やったことにしたい場合は、まず今日やるにしてから、今日やる画面でチェックする。

## 今日やる画面と今後やる画面の共通化

今日やる画面と今後やる画面は、できるだけ同じUI・同じDOM・同じ関数で実装する。

共通にするもの:

- タスク行
- タスク追加フォーム
- 編集
- 削除
- タグ表示
- 日付変更
- 小さなステップの表示・編集

モードごとに変えるもの:

```tsx
<TaskListView mode="today" />
<TaskListView mode="future" />
```

today モード:

- 今日やる条件で絞り込む
- 毎日タスクを表示する
- チェックボックスを表示する
- 完了ログを作る
- 追加時の `scheduledDate` は今日

future モード:

- 今後やる条件で絞り込む
- 毎日タスクは表示しない
- チェックボックスは表示しない
- 完了ログは作らない
- 追加時に日付未定または日付指定を選べる

## カレンダー画面

カレンダーは、予定と記録を閲覧する場所。

日付を押すと、その日の情報を表示する。

表示するもの:

- その日に予定されている未完了タスク
- その日にできたタスク
- その日の「ひとこと」

カレンダー上の `+` から、その日に予定されたタスクを追加できてもよい。
この場合、追加されるタスクの `scheduledDate` は選択中の日付になる。

## ひとこと

日ごとの文章は「メモ」ではなく「ひとこと」と呼ぶ。

理由:

- タスクと混同しにくい
- カレンダーの日付に紐づく短い記録として扱いやすい

既存の `day_notes` は、UI上では「ひとこと」と表示する。

## 完了ログ

完了ログは、実際にチェックした日付で記録する。

```txt
doneLog.date = チェックした日
scheduledDate = 予定日
```

完了操作は今日やる画面でのみ行う。
