## 考え方（いちばん大事）

概念はできるだけ増やさない。**すべては「アイテム（やること）」ひとつ。**

- **毎日くりかえすか?** → トグルで切り替え（オン＝習慣、今日タブに自動で出る）
- **どんなテーマか?** → タグ。`#裁縫` のように書くと付く。**タグの管理画面は作らない**
- タスク / 習慣 / ストック / カテゴリ という区分は、ぜんぶこのモデルに統合した

---

## 今できること（この時点で完成している部分）

AI なしで、もう毎日使える状態です。

- **今日タブ**：毎日の習慣が自動で並ぶ＋今日やることを選ぶ → 手順を手で足す → ✓ でチェック →「できた」
  - まだ終わっていない最初の手順に「次の一歩」の印が付く
- **一覧タブ**：アイテムを追加・編集・削除。`#タグ` を書いて付ける。`#` を打つと既存タグの候補が出る。「毎日」トグルで習慣に。タグで絞り込み（＝買い物リスト等はタグ絞り込みで再現）
- **カレンダータブ**：月表示。できた数が見える。日付を押すとその日の「できたこと」一覧。メモも足せる
- データは **Supabase に保存**。ログインすれば **スマホ⇄PC で同期**できる。「できたこと」は元アイテムを消しても残る

## データの形（src/types.ts）

- **Item** … やること。`title` / `tags[]` / `recurring`(毎日か) / `status` / 日時
- **Step** … 手順。`itemId` でどのアイテムの手順かを表す
- **TodayItem** … 今日やると選んだアイテム（毎日の習慣は選ばなくても自動で出る）
- **DoneLog** … できたことの記録（title をコピーして持つので消えない）

保存と操作は `src/store.ts` に集約。画面は `useStore()` で最新データを読むだけ。

### データの保存（Supabase）

- 保存先は **Supabase**（Postgres）。ログインしたユーザー自身のデータだけ読み書きできる（RLS で保護）。
- 仕組み：メモリ上に最新データを持ち、変更は「メモリを即更新（楽観更新）＋裏で Supabase に書き込み」。読み込みはログイン時に `hydrate()` でまとめて取得。
- `src/supabase.ts` … クライアント（URL/anon キーは `.env.local`）
- `src/db.ts` … Supabase の読み書き（TS の camelCase ↔ DB の snake_case 変換もここ）
- `supabase/schema.sql` … テーブル定義＋RLS。**Supabase の SQL Editor に貼って実行する**
- 初回ログイン時、データが空なら、以前 localStorage にあった分 or `seedDB()`（Any.do の初期データ）をアップロードする（`seedIfEmpty()`）。
- Any.do 由来データと修正（川口プラネタリウム→行きたい場所、すりっぱー→見る・読む）は `src/seed.ts`。

### ログイン

- メールにログイン用リンクを送る方式（マジックリンク、パスワード不要）。`src/screens/LoginScreen.tsx`。
- `App.tsx` がログイン状態を監視し、未ログインならログイン画面、ログイン後にデータを読み込んで本体を表示。

### セットアップ（新しい環境で動かすとき）

1. `.env.example` をコピーして `.env.local` を作り、Supabase の URL と anon キーを入れる
2. `supabase/schema.sql` を Supabase の SQL Editor で実行
3. `npm install` → `npm run dev`
