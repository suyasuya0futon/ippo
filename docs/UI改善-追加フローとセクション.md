# UI改善：追加フローとセクション設計（叩き台）

> ⚠️ **参考。** 最新の正は [設計-今後やる刷新（フラグ・予定日・行UI）.md](./設計-今後やる刷新（フラグ・予定日・行UI）.md)。
> 「➕で各セクションに追加」の作法はこの文書が出どころで現行も踏襲。ただし詳細は最新の正を見ること。

> 構造リファクタ後の「見た目・操作」磨きパス。最初のテーマは「追加」と「セクション」。

## 原則

1. **常時表示の「タスクを追加」フォームは廃止。** 画面の一番上に居座らせない。
2. **各セクションの見出しに ➕ を置く。押したときだけ入力欄が出る。**
3. **追加するものの種類は「どのセクションの ➕ か」で決まる。**
   → 「毎日くりかえす」チェックボックスは廃止（文脈で自明）。
4. 入力欄には今までどおり `#タグ` の候補表示を出す。

## 見出しの形

```
毎日の習慣        3/10達成        ➕
今日のタスク      0/3達成         ➕
今後やる                         ➕
```

- 左：セクション名
- 中：達成数（今日やる側だけ。全部できたら 🎉）
- 右：➕（押すと、その見出しの直下に入力欄が開く）

## 今日やる

- **毎日の習慣** 見出し（`X/Y達成` ＋ ➕）
  - ➕ → 習慣の入力欄（`recurring = true` で追加。チェックボックスなし）
- **今日のタスク** 見出し（`X/Y達成` ＋ ➕）
  - ➕ → 今日のタスク入力欄（`scheduledDate = 今日` で追加）

## 今後やる

- **今後やる** 見出し（➕）
  - ➕ → 未定タスクの入力欄（`scheduledDate = null` で追加）
  - 達成数は出さない（完了操作をしないため）

## カレンダー（この日の予定）

- 「この日に予定を追加」も、常時入力ではなく見出しの ➕ に寄せてよい（同じ作法）。
  - ➕ → `scheduledDate = 選択日` で追加

---

## 決定事項

1. **入力欄は1件追加したら閉じる**（開きっぱなしにしない）。
2. **「今日やることを選ぶ」ピッカーは廃止**（表示も内部処理もまるごと削除）。今日に出したいものは今後やるタブの 🌱「今日やるにする」で。
3. **編集で種類（習慣⇄タスク）は変えない**。習慣にしたいなら習慣セクションの ➕ から登録する。編集はタイトル/タグのみ。**「毎日くりかえす」チェックボックスは全廃**。
4. **達成数の文言は今のまま**（`3/10達成` / `10/10達成🎉`）。将来もっと派手にするかも。
5. **➕ は絵文字の ➕**（丸いアイコンボタンではない）。
6. **➕ はセクション見出しと同じ行・右端に右寄せ。改行させない**（`flex-shrink: 0`）。

実装順：今日やる → 今後やる → カレンダー。各セクションの ➕ 化を進める。
