# 設計：予定日の廃止 ＋ カレンダー →「できた帳」

> [設計-今後やる刷新（フラグ・予定日・行UI）.md](./設計-今後やる刷新（フラグ・予定日・行UI）.md) のうち「予定日(scheduledDate)」に関する部分は廃止。この文書で置き換える。

## 1. 予定日（scheduledDate）を廃止

- 「いつやるか」は **フラグ(bucket: today/tomorrow/soon/someday) だけ**にする。特定日のピン留めは持たない。
- 特定日の約束（病院など）は IPPO では扱わない（必要なら別カレンダーで）。
- 影響：
  - `effectiveBucket` は不要 → 配置は `item.bucket` をそのまま見る。
  - `setScheduledDate` / addItem の scheduledDate オプション / 日付UI（ステップ6）を削除。
  - 🌱を「予定日つきには出さない」みたいな例外が消える（全タスクがフラグ）。
  - ドラッグ（ステップ7）も「予定日つきは動かさない」分岐が消えて軽くなる。
  - DBの `scheduled_date` 列は使わなくなる（列は残置でOK。後でドロップしても良い）。
- 「できたこと(done_logs)」は予定日と無関係 → **完全に無傷**。

## 2. カレンダータブ →「できた帳」

「やったことを振り返る帳面」。内部は月カレンダーではなく **一覧**。

### 表示
- **タグ絞込**（上部にチップ：すべて／各タグ）。選ぶと、そのタグの「できたこと」だけに絞る。
  - 例：`#病院` を選ぶ → 病院に行った記録が日付つきで並ぶ →「前回いつ?」が分かる。
- **日付ごとにまとめた一覧（新しい日が上）**。各日のかたまり：
  - 日付見出し（例：2026年6月19日（金））
  - **できたこと**（その日の done ログ：時刻 ＋ タグ ＋ タイトル）

### やめるもの
- 月カレンダーのグリッド／月送り
- 「この日の予定」セクション（予定日廃止のため）
- 日付つきタスク追加
- ひとこと

### 残すもの
- done_logs（できたこと。title/tag をコピーして持つので元タスクを消しても残る）

### データ
- done_logs は今のまま（tag を持っているのでタグ絞込できる）。
- day_notes はアプリコードから読まない・書かない。DBテーブルは残置でOK（後でドロップしても良い）。

## 3. 実装の段取り（案）

1. **予定日撤去**：型・store・db・TaskListView から scheduledDate を外す（`effectiveBucket`→`bucket`）。CalendarScreen の予定セクションも一旦外す。
2. **できた帳に作り替え**：CalendarScreen を done_logs の一覧（日付ごと＋できたこと）＋タグ絞込に。タブ名「できた帳」、アイコンは 📖 あたり。
3. （別途）ステップ7 ドラッグ。

ステップ6（日付UI）は**廃止**。

2. done が増えたとき、まずは**全部表示**でいい?（重くなったら「もっと見る」を後で）
