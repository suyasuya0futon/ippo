## コンセプト

- 「いつやるか」は普段 **ゆるいフラグ**：`今日 / 明日 / 近日中 / いつか`。日付は持たない。
- **予定日（scheduledDate）** は特別＝病院・約束など「その日」が決まってる時だけ入れる。**予定日つきだけカレンダーに出る。**
- **並び順は手動。自動で並び替えない。** 追加は常に **先頭（いちばん上）** に出る。順番は保持する。
- フラグは時間で自動的に繰り上がる（明日→今日／近日中→明日）。予定日つきは日付で自然に近づく。
- 行はスッキリ：**タップで編集パネルが開く（インラインで展開）**。鉛筆アイコンは無し。

---

## データ

```ts
Item {
  id, title, tag, recurring, status, createdAt, doneAt?
  bucket: "today" | "tomorrow" | "soon" | "someday"  // ゆるいフラグ（予定日なしの時の配置）
  scheduledDate: string | null                        // 予定日（任意・約束）。あればカレンダー表示＆自動で近づく
  sortOrder: number                                   // 手動の並び順（バケット内）。小さいほど上。
}
```

- `scheduledDate != null` → 約束。カレンダーに出る。バケット配置は **日付から導出**（自動で流れる）。
- `scheduledDate == null` → `bucket` のフラグで配置。カレンダーには出ない。
- `recurring`（毎日タスク）は従来どおり（今日やるに自動表示、完了は日ごとログ。bucket/scheduledDate/sortOrder は使わない）。

---

## どこに表示されるか

### 今日やる（タブ）
- `bucket === "today"` のフラグタスク
- 予定日つきで `scheduledDate <= 今日`（繰り越し含む）
- 今日完了したもの（打ち消し線で残す）
- 毎日の習慣

### 今後やる（タブ）＝ 明日 / 近日中 / いつか（アコーディオン・全部開く）
- **明日**：`bucket === "tomorrow"` ∪ 予定日 == 明日
- **近日中**：`bucket === "soon"` ∪ 予定日が明後日以降（未来日は全部ここ。予定日チップ付き）
- **いつか**：`bucket === "someday"`（予定日なしのみ）
- 各バケット内は **sortOrder 順（手動）**。日付やタグでの自動ソートはしない。
- 見出し：**左（名前＋件数）= 開閉スイッチ / 右端 ➕ = そのバケットへ追加**。

### カレンダー
- **予定日つきのタスクだけ** その日に表示。フラグタスクは出さない。
- 予定欄は **未完了のみ**。完了したものは「できたこと」に出る（予定欄には残さない）。

---

## 並び順（手動）

- 各バケットは手動の並び（`sortOrder`）。**自動ソートしない。**
- **追加は先頭**（新しいものがいちばん上）。
- 並べ替えは **ドラッグ**（後のステップ）で。ドラッグで挿入位置に応じて sortOrder を更新。
- バケットをまたいでドラッグ → 移動先バケット（フラグ）に変更＋移動先での順番を持つ。

---

## 自動の繰り上げ（promote）

起動時に「前回から日付が変わったか」を見てフラグを書き換える（0時ちょうどでなく、日跨ぎ後の初回起動で。何日空けても1回だけ＝gap安全）。

- **日が変わった**：`bucket "tomorrow" → "today"`
- **週が変わった（月曜をまたいだ）**：`bucket "soon" → "tomorrow"`（一括。Any.do の“週替わりでドサッ”を踏襲）
- 同じ起動で両方なら「tomorrow→today」→「soon→tomorrow」の順。
- `"someday"` は自動では動かない。`"today"` はそのまま（未完了は繰り越し）。
- 予定日つきは mutate せず毎回 日付から導出。
- `lastPromoteDate` は **Supabase に user 単位で保存**（例：`ippo.user_settings(user_id pk, last_promote_date date)`）。
  - localStorage は使わない（前に消した混入リスクの再発防止）。
  - 共有保存なので、複数デバイスでも**二重繰り上げを防げる**（先に開いた端末が更新→他端末は同週なら繰り上げない）。
- 繰り上がったものは移動先バケットの **末尾（いちばん下）** に積む（確定）。
  - 手動の並びを荒らさないため。役割分担：**手動追加＝先頭／自動繰り上がり＝末尾**。

## 行UI（Any.do 寄せ）

1行はスッキリ。常時出すのは最小限：

- 左：**チェック○**（今日やるのときだけ。完了/取り消し）
- 中：**タグ ＋ タイトル**
- 右：
  - **移動ボタン（独立・背景なし。タブのアイコンと向きを揃える）**
    - 今後やる → **🌱「今日やるにする」**（bucket=today）
    - 今日やる → **⏳「今後やるに移動」**（bucket=**soon／近日中**。「いつか」送りで埋もれさせない）
    - **このボタンはフラグタスク（予定日なし）だけに出す。予定日つきには出さない。**
      予定日つきを動かすと約束の日を壊す事故になるため。予定日つきの移動は編集パネルで日付を変える/外す。
    - 「今日に居る印」の緑背景は廃止（向きアイコンで意味が出る）。
  - **クイック ×**（**完了済みのアイテムにだけ**表示。サッと片付け用）
- **鉛筆アイコンは無し。**

### タップで編集パネル（インライン展開）
- タスク行（タイトル付近）を**タップ**すると、その場で**アコーディオンが開いて編集パネル**が出る（もう一度タップ/別の所で閉じる）。
- パネルの中身：
  - **タイトル編集**（＋#タグ）
  - **予定日**（📅。設定／変更／なし）
  - **削除**（常設。今後やるのタスクもここから消せる）
- **バケット切り替えはパネルではやらない**（移動＝ドラッグ／🌱 で行う）。
  - ⚠️ **暫定の制限**：ドラッグは後のステップなので、それまで 明日↔近日中↔いつか を動かす手段が無い。
    **間違えたら削除して作り直す**運用とする（本番未稼働なので許容。ドラッグ実装で解消）。
- チェック○のタップは「完了」、それ以外のタップは「編集を開く」。ターゲットを分けて競合させない。

### 削除のまとめ
- 編集パネル内に**常設**（誤削除しにくい＝一手間ある）。
- 完了済みアイテムには加えて**クイック ×**。

## 追加フロー（➕）

- 各セクション見出しの右端 ➕ → 入力欄がインラインで開く（1件追加で閉じる、チェックボックス無し）。
- 追加先のフラグはセクションで決まる：
  - 今日やる「今日のタスク」➕ → `bucket="today"`
  - 今後やる「明日」➕ → `bucket="tomorrow"` ／「近日中」➕ → `bucket="soon"` ／「いつか」➕ → `bucket="someday"`
  - カレンダー「この日に予定」➕ → `scheduledDate = その日`（予定日つき）
  - 今日やる「毎日の習慣」➕ → 習慣
- 追加されたものは **そのバケットの先頭**。

## 予定日の付け外し
- 付ける → 約束になりカレンダーに出る。配置は日付導出。
- 外す → フラグに戻る。**戻り先は「いつか」**（確定）。

---

## ドラッグ移動（後のステップ）

- タッチ対応の並べ替え＝**dnd-kit** 導入想定。重いので、フラグ＋予定日＋手動順が動いてから別ステップ。
- **フラグタスク**：バケット内＝並べ替え（sortOrder 更新）／バケット間＝移動先フラグに変更＋順番更新。
- **予定日つきタスク**：居場所は“日付”が持ち主なので **バケット間のドラッグ対象にしない**（同じバケット内の並べ替えだけ）。
  - 動かす＝編集パネルで **予定日を変更／「日付なし」で外す**（＝意図した操作のときだけ）。
  - これで「わざわざ入れた予定日がドラッグでうっかり消える/別の日になる」を防ぐ＝約束を守る。

---

## 既存データの移行（確定）

今の `scheduledDate` は today_items 移行で自動的に入った日付が多く本物の約束ではない → 全部フラグへ変換し予定日はクリア。

- `scheduledDate == null` → `bucket="someday"`
- `scheduledDate <= 今日` → `bucket="today"`、予定日クリア
- `scheduledDate == 明日` → `bucket="tomorrow"`、予定日クリア
- `scheduledDate > 明日` → `bucket="soon"`、予定日クリア
- `sortOrder` は createdAt の新しい順などで初期付与（バケット内）。
- 毎日タスクはそのまま。
- DB：`bucket text` と `sort_order` 列を追加。`scheduled_date` は残す（意味が「約束の日」に変わる）。

## 実装計画（案・一歩ずつ）

1. **モデル追加**：`bucket` / `sort_order` 列＆型を追加（migration）。既存データを上の規則で変換。
2. **配置ロジック差し替え**：今日やる/今後やるの抽出を bucket ベースに。今後やるを 明日/近日中/いつか の3バケット＋アコーディオンに。並びは sortOrder（自動ソート撤去）。➕でフラグ付与・先頭追加。
3. **繰り上げ promote**：起動時の日跨ぎ/週跨ぎでフラグ書き換え（lastPromoteDate）。
4. **カレンダー**：予定日つきだけ表示に。
5. **行UI**：鉛筆廃止／タップで編集パネル（タイトル・予定日・削除）／完了済みにクイック×／🌱は独立。
6. **日付UI（A案）**：予定日はチップ＋showPicker（編集パネル内）。
7. **（別）ドラッグ**：dnd-kit で並べ替え＆バケット移動。

## 決まったこと（細かい点）
- 繰り上がったフラグは移動先の **末尾**（手動の並びを荒らさない）。
- 予定日つきは **ドラッグでバケット移動させない**（日付が持ち主）。移動は編集パネルで日付を変更／外す。予定日はドラッグでは消えない。
- **移動ボタンはフラグタスクのみ**に表示（予定日つきには出さない＝約束を壊さない）。
  - 今後やる＝🌱「今日やるにする」(→today)／今日やる＝⏳「今後やるに移動」(→**soon/近日中**)。緑背景は廃止。
- **`last_promote_date` は Supabase（`ippo.user_settings`）に保存**。localStorage 不使用。
- **予定日つきの sortOrder**：追加/予定日設定時は、その表示バケットの **先頭** に入れる（手動追加と同じ＝上）。
- **カレンダーの予定欄は未完了のみ**（完了は「できたこと」へ）。
- **編集パネルにバケット切替は入れない**（B案）。ドラッグ実装までは「間違えたら作り直し」。（ドラッグ実装時に確定でも可）
