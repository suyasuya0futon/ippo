## 1. 日付UI（A案で確定）

- 生の `<input type="date">` を全行に常時表示するのをやめる。
- **未定の行**：日付UIは出さない（アクション群に小さな 📅 ボタンだけ）。
- **日付ありの行**：タイトル隣に小さなチップ **`📅 6/23`**（タップで変更、チップの ✕ で解除）。
- 日付選択は 📅 を押した時に**ネイティブのカレンダーをその場で開く**（`showPicker()`。生の入力ボックスは画面に出さない）。
- チップの表記は **`6/23`**（年は省略。年をまたぐ先のものだけ `2027/1/5` のように年を出す案もあり→要検討）。

---

## 2. 並び順

- 今後やるは **予定日が近い順（昇順）に上から**。
- 日付なし（いつか）は最後。
- 同じ日・同じバケット内はタグ降順→新しい順（今の byTag 準拠）。

---

## 3. バケット（時間グループ）＋アコーディオン

今後やるを、時間のかたまりで見出し分けして表示する。各見出しは**アコーディオン（たためる）**。

### バケット案（4つ）

| バケット | 条件 |
|---|---|
| **明日** | 予定日 == 明日 |
| **今週** | 明後日 〜 今度の日曜まで |
| **それ以降** | 来週月曜以降の日付つき |
| **いつか** | 日付なし |

- 各バケット見出しに件数（例：`今週 (3)`）と開閉の ▸/▾。
- 空のバケットは見出しごと隠す。
- 既定の開閉：**明日・今週は開、それ以降・いつかは閉**（要検討）。

### 判断ポイント

1. あなたは「明日 / 今週 / いつか」の3つと言っていた。**「それ以降」を足して4つ**にする?
   それとも「来週以降の日付つき」も**「いつか」にまとめて**3つにする?
   （私のおすすめ：4つ。日付つきを「いつか」に混ぜると日付情報が埋もれるため）
2. **週の区切り**：「今週」は今度の**日曜**まで、で良い?（日曜終わり想定）
3. 既定の開閉は上の案でいい?

---

## 4. ドラッグで移動（Any.do 的）

タスクを指で長押し→上下にドラッグして、別のバケットへ移せる（例：今週→明日）。
ドロップ先のバケットに応じて **`scheduledDate` を書き換える**。

### ここが重い・要設計

- **タッチ対応の並べ替え**は自前だと大変。ライブラリ **dnd-kit** を入れるのが現実的（タッチ＋グループ間移動に対応）。依存が1つ増える。
- **範囲バケットに落とした時の日付**が曖昧：
  - → **明日**：`scheduledDate = 明日`（確定）
  - → **いつか**：`scheduledDate = null`（確定）
  - → **今週**：範囲なので日付が一意に決まらない。案：**今度の日曜**にする / **今日+2日**にする / ドロップ位置の前後の日付の間にする（複雑）
  - → **それ以降**：案：**来週月曜**にする
- 並べ替えで「同じバケット内の順番」を持たせるなら、`sortOrder` 的な列が要る（今は日付とタグでしか並ばない）。バケット内手動並べ替えまでやるとさらに重い。

### 方針案（おすすめの進め方）

1. **先に 1〜3（日付チップ＋並び＋バケット＋アコーディオン）を実装**。これだけでかなり使いやすくなる。
2. ドラッグは別ステップで。まずは**「ドロップ先＝明日 / いつか」のような“日付が一意に決まる移動”から**始め、範囲バケットの扱いは様子を見て決める。
   - もしくは、ドラッグの代わりに**チップの📅から日付変更**で実用上は足りるかも（その場合ドラッグは後回し or 不要）。

### 判断ポイント

4. ドラッグは**今回やる**? それとも**1〜3を先に出して、ドラッグは次のステップ**にする?
5. ドラッグやるなら、**範囲バケット（今週/それ以降）に落とした時の日付ルール**はどれにする?

## 決定事項（確定）

**バケットは3つ：明日 / 今週 / いつか**（Any.do の Tomorrow/Upcoming/Someday 相当。Today は別タブ）

| バケット | 条件 |
|---|---|
| **明日** | 予定日 == 明日 |
| **今週** | 明後日 〜 今度の日曜 |
| **いつか** | 日付なし ＋ 来週以降（今度の日曜より先）の日付つき |

- 「今週」の区切りは**今度の日曜**まで。
- 来週以降の日付つきは**「いつか」に入れる**が、**日付チップで日付を見せ、いつか内では日付つきを上に（日付順）→日付なしを下**に並べる（日付は埋もれない）。
- 並びは全体として予定日の近い順。

**アコーディオン**
- 既定は**全部開く**。
- 見出しは「**左の部分（バケット名＋件数）= 開閉スイッチ**」「**右端の ➕ = そのバケットへ予定を登録**」。タップ領域を分けて競合させない。

**各バケットの ➕ で追加したときの予定日**（ドラッグのドロップ時も同じルール）
- → 明日：`scheduledDate = 明日`
- → 今週：`scheduledDate = 今度の日曜`
  - ※当初案の「今週あたまの月曜」は、週の途中だと**過去日になって今日やるに落ちる**ので不採用。今週内に確実に収まる**今度の日曜**にする。
- → いつか：`scheduledDate = null`

**日付が変わったとき（0時）**
- 今日やるは `scheduledDate ≤ 今日` 判定なので、日付が変われば「明日」のタスクが自動的に今日やるへ。
- ただしアプリを開いたまま0時を跨いでも即時には動かない（再描画時/開き直しで反映）。必要なら0時更新タイマーを足す（今は不要）。

**実装順**
1. 日付UI（A案：チップ＋showPicker）
2. 今後やるを予定日の近い順に並べる
3. 3バケット＋アコーディオン＋バケットごとの ➕
4. （別ステップ）ドラッグ移動（dnd-kit。ドロップ時の日付は上のルール）

ドラッグは重い（タッチ対応の並べ替え＝dnd-kit 導入）ので、**まず 1〜3 を実装し、ドラッグは次のステップ**にする。
