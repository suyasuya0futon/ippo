# データモデル改定：フラグ（いつやる）＋ 任意の予定日

> ⚠️ **旧案・参考。** 最新の正は [設計-今後やる刷新（フラグ・予定日・行UI）.md](./設計-今後やる刷新（フラグ・予定日・行UI）.md)。
> こちらは検討途中のメモで、現行設計と細部が異なる（手動順・🌱の条件・保存場所など）。

> 「全部を日付で管理する」のをやめる。普段は **ゆるいフラグ**（今日/明日/近日中/いつか）。
> 特定の日が決まっている約束だけ **予定日** を入れる。予定日つきだけカレンダーに出る。

---

## コンセプト（考え方）

- 大半のタスクは **フラグ** で「だいたいいつやるか」を持つ。日付は持たない。
- **予定日（scheduledDate）** は特別なもの＝病院・約束など「その日」が決まっているとき**だけ**入れる。
- フラグは **時間で自動的に繰り上がる**（明日→今日、近日中→明日）。予定日つきは日付で自動的に近づいてくる。
- **カレンダーには予定日つきのタスクだけ**出る（フラグだけのものは出ない）。

---

## データ

```ts
Item {
  id, title, tag, recurring, status, createdAt, doneAt?
  bucket: "today" | "tomorrow" | "soon" | "someday"  // ゆるいフラグ
  scheduledDate: string | null                        // 予定日（任意・約束）
}
```

- **予定日が入っているか / フラグか は排他**で考える：
  - `scheduledDate != null` → 予定日つき（＝約束）。配置は日付から導出。カレンダーに出る。`bucket` は無視。
  - `scheduledDate == null` → フラグ式。`bucket` で配置。カレンダーに出ない。
- 毎日タスク（`recurring`）は今までどおり。`bucket`/`scheduledDate` は使わない（今日やるに自動表示、完了は日ごとログ）。

---

## どこに表示されるか

### 今日やる（タブ）
- `bucket === "today"` のフラグタスク
- 予定日つきで `scheduledDate <= 今日`（繰り越し含む）
- 今日完了したもの（打ち消し線で残す）
- 毎日の習慣

### 今後やる（タブ）＝ 明日 / 近日中 / いつか
- **明日**：`bucket === "tomorrow"` ∪ 予定日 == 明日
- **近日中**：`bucket === "soon"` ∪ 予定日が明後日以降（＝今日・明日以外の未来日は全部ここ。日付チップ付きで日付順に上へ）
- **いつか**：`bucket === "someday"`（予定日なしのみ）

### カレンダー
- **予定日つきのタスクだけ**、その日に表示。フラグタスクは出さない。

---

## 自動の繰り上げ（promote）

アプリ起動時に「前回から日付が変わったか」を見て、フラグを繰り上げる（0時ちょうどでなく、日跨ぎ後の初回起動で実行。何日空けても1回だけ）。

- **日が変わった**：`bucket "tomorrow" → "today"`
- **週が変わった（月曜をまたいだ）**：`bucket "soon" → "tomorrow"`
  - 同じ起動で両方起きる場合は「tomorrow→today」→「soon→tomorrow」の順（近日中だったものは“明日”止まりで、いきなり今日には飛ばさない）
- `"someday"` は自動では動かない（手動のみ）
- `"today"` はそのまま（未完了なら翌日も今日やるに残る＝繰り越し）
- 予定日つきは mutate せず、毎回 日付から配置を導出（自然に近づく）

実装：`lastPromoteDate`（最後に繰り上げた日付）を保存。起動時に今日と比較して必要分だけ実行。

## 追加フロー（➕）

- 今日やる「今日のタスク」➕ → `bucket = "today"`
- 今後やる「明日」➕ → `bucket = "tomorrow"`
- 今後やる「近日中」➕ → `bucket = "soon"`
- 今後やる「いつか」➕ → `bucket = "someday"`
- カレンダー「この日に予定」➕ → `scheduledDate = その日`（＝予定日つき）
- 今日やる「毎日の習慣」➕ → 習慣

## 予定日の付け外し

- フラグタスクに予定日を付ける → 予定日つき（約束）になり、カレンダーに出る。`bucket` は無視。
- 予定日を外す（日付なし）→ フラグに戻る。**戻り先のフラグは「いつか」**（要確認。または外した時点の配置から推定）。

---

## ドラッグ移動（後のステップ）

- バケット間でドラッグ → そのバケットのフラグに変更（today/tomorrow/soon/someday）。
- 予定日つきをフラグバケットへドラッグ → 予定日を外してフラグに（要確認）。
- 重い（タッチ対応＝dnd-kit）ので、フラグ＋予定日モデルが動いてから別ステップで。

---

## 既存データの移行（要確認）

今の `scheduledDate` は、today_items 移行で**自動的に入った日付**が多く、本物の約束ではない。なのでカレンダーに出てしまうと不自然。移行案：

- `scheduledDate == null` → `bucket = "someday"`
- `scheduledDate <= 今日` → `bucket = "today"`、予定日はクリア
- `scheduledDate == 明日` → `bucket = "tomorrow"`、予定日はクリア
- `scheduledDate > 明日` → `bucket = "soon"`、予定日はクリア
- （＝既存の日付は全部フラグへ変換し、予定日つきは一旦ゼロにする。本物の約束は後から付け直す）
- 毎日タスクはそのまま。

## 判断ポイント

1. **近日中 → 明日 は週替わり（月曜）に一括**でOK（確定）。
2. 予定日を**外したとき**の戻り先フラグは「いつか」でいい?
3. 既存データの移行は上の案（既存日付は全部フラグ化・予定日クリア）でいい?
4. DBは `bucket text` 列を追加。`scheduled_date` は残す（意味が「約束の日」に変わる）。これでいい?
